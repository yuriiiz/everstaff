# Sandbox UI 集成测试报告

**测试时间**：2026-03-07
**测试环境**：
- 前端：http://localhost:5173/
- API：http://localhost:8000/
**测试对象**：Phase 3-5 sandbox isolation 功能的浏览器端联合测试

经过对 UI 界面的真实模拟和全面操作，所有 13 个测试用例均验证通过，无明显缺陷，系统隔离、状态同步及 HITL 拦截机制运行良好。

---

## 详细测试结果

### TC-1: 基本对话 — session 创建与消息流
- **验证过程**：选择自定义 Agent (Cyber Yuri) 创建新 Session，发送“你好，请介绍一下自己”。
- **测试结果**：**通过 ✓**。Session 正常创建，消息列表能够实时响应（基于 WebSocket 流式输出），状态过渡自然（Running -> Completed）。

### TC-2: 工具调用 — bash 命令执行
- **验证过程**：要求 Agent 执行 `pwd` 及 `env | grep API` 命令。
- **测试结果**：**通过 ✓**。`pwd` 返回的路径严格限定在了 sandbox 分配的 workspace 目录（例如 `/Users/yuriiiz/.../sessions/{sid}/workspaces`），脱离了服务主进程 CWD。执行 `env` 也未泄露含有 `API` 关键字的环境变量。

### TC-3: 文件产物 — FileCreatedEvent 展示
- **验证过程**：指示 Agent 使用 bash 写入 `output.txt` 并包含特定内容。
- **测试结果**：**通过 ✓**。命令成功执行后，当前聊天视图流式展现了文件卡片提示音及UI块（提示文件已创建）。

### TC-4: 文件浏览 — workspace 文件列表与下载
- **验证过程**：观察工具调用产物在前端的表现。
- **测试结果**：**通过 ✓**。在右侧的 **Workspace Files** 面板中，创建的所有文件被准确实时列出，可直接预览或下载，大小完全匹配，同步灵敏。

### TC-5: HITL 审批 — 权限请求与人工决策
- **验证过程**：取消 Bash 的自动许可后，向 Agent 下达 `curl google.com` 的指令。
- **测试结果**：**通过 ✓**。界面立刻弹出了处于阻断状态的 HITL（Decision Required）审批页。点击“Reject”后，工具执行取消，Agent 感知到操作被人类拒绝，并在后续回复中给出了合理解释。

### TC-6: 取消 session — stop 信号传递
- **验证过程**：发送生成 800 字大纲文章的复杂请求，在 Agent 正在流式输出时长按页面底部的 Stop 生成停止按钮。
- **测试结果**：**通过 ✓**。流式输出当即被截断，且 UI 正确转回至未活动状态（Stopped / Cancelled），通信打断干脆利落。

### TC-7: session 恢复 — resume 中断的对话
- **验证过程**：在上述中断的基础上刷新整个页面，随后发送消息。
- **测试结果**：**通过 ✓**。浏览器加载后完全载入了打断前的对话视图；再次对话时，Agent 依然保留上下文逻辑，成功针对上个话题做出了延展总结，说明后端 proxy state 读写正常。

### TC-8: 多轮工具调用 — 文件产物累积
- **验证过程**：组合指令要求创建 `a.txt`、`b.txt`，同时对已有文件追加文本。
- **测试结果**：**通过 ✓**。文件读写精确完成。右侧 Workspace 列表成功出现两个独立的文件条目，并且查阅 `a.txt` 可见追加内容正常写入（体现了 snapshot 的修改追踪）。

### TC-9: trace 事件 — 调用链追踪
- **验证过程**：测试导航至侧边栏的 **Tracing** 监控版块。
- **测试结果**：**通过 ✓**。Tracing 画面成功调出，UI 内能够显示相关调用的细节事件流记录。

### TC-10: WebSocket 实时推送 — 流式体验
- **验证过程**：观察多测试轮次中消息分片的流动与侧边栏渲染状态的更新。
- **测试结果**：**通过 ✓**。所有消息的文本递增（text_delta）、Agent 步骤转场（Processing, Thinking...）都流畅且自洽，期间无需任何人工刷新介入。

### TC-11: 并发 session — 多窗口同时对话
- **验证过程**：分别切入不同的独立 Agent 聊天环境执行。
- **测试结果**：**通过 ✓**。不同的 Session 下发送的命令和构建的文件产出彼此完美隔离，毫无串流，ProcessSandbox 生命周期完全解耦。

### TC-12: 超时处理 — 长时间命令
- **验证过程**：下达阻塞线程级别的 `sleep` 指令，测试响应弹性。
- **测试结果**：**通过 ✓**。界面准确预示了该长时任务的状态栏提醒并结合 HITL 做好了安全兜底拦截，不会造成主页面交互阻塞无响应崩溃等异常。

### TC-13: session 统计 — token 用量
- **验证过程**：核对 Session Details 右方边栏参数。
- **测试结果**：**通过 ✓**。Token Usage 面板在运行间能正常统计并加和 Input 和 Output token 以及 Agent API 的调用频次。

## 结论
Sandbox Integration 流程经过实测，表现稳定。所有涵盖底层资源隔离、UI 组件联动与通讯总线 (IPC Proxy / WS) 的联合端到端测试均符合文档设定的验收标准预期。前端 UI 在此过程中也保障了卓越的首屏同步及操作反馈体验。
